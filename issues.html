Feedback for the current Digital Skills page - Learning Company website

Functional issues

  • a booking button has been reported to not work, though I have not tried to reproduce this issue
problem justification solution
when the browser window is snapped to the side (narrow), 2 sections ("Why Learn with our Learning Company?" & "Our Digital Learning Solutions") are not shown. these sections are also missing from the mobile website 70% of webpages are viewed on mobile devices. missing sections will likely confuse and frustrate the user, reducing the likelihood that they will engage. this issue is most likely cause by the way different (incompatible) Wix elements interfere with each other change the code for responsiveness (write from scratch or check the behaviour of the Wix elements)

Logical/flow issues

flow

problem justification solution
unintuitive flow of sections as a potential client, I want to read the page in this order: why should I consider a course from BLC? -> what courses do they offer? -> do they have this course for my skill level? re-order the sections
the skills section disrupts the flow of the page it is bigger than the other sections so it takes longer to scroll past if the user is not interested in reading it reduce its length to match the other sections, either by collapsing it or implementing horizontal scrolling. I used the design of the apple store page for inspiration for the scrolling part

redundancy & irrelevance

problem justification solution
the digital skills page has a contact form identical to the one on the contact page the form is not relevant to booking a session, users may find it confusing when trying to book a session replace it with a session booking form
booking buttons lead to an email message with a specific subject line instead of a booking form you rely on the user to provide all the information you need to book a course in one email, as they don't have a template to fill out. email buttons are useful for enquiries, but less useful for booking a form structured to record information about the client, course, skill level, date & time, and other relevant information in a structured and easy to read manner, without the need to go back an forth with the client for more information
"Why Learn with our Learning Company?" has a button that says "Book" it does not provide information about what course the user is booking by clicking on the button, as each skill course already has its own booking button remove this button
each course has its own booking button Hick's law violation: overwhelming number of choices for the user, when in reality all the buttons do the exact same thing remove all the booking buttons under the course cards and direct the user to the singular booking form at the bottom of the page through the natural reading flow. I also added the ability for the user to skip to it from the top of the page by clicking on the "Book Now" button in the hero element as is common for CTAs in other websites
the hero (first) section has a lot of text, and the words in the "Whether you are looking for introductory, intermediate or advanced training" part are repeated in a later section (the skill level section) the hero section text should be snappy and to the point to hook the user in, and it is unnecessary to repeat information that user will read several seconds later in a following section remove that phrase from the hero text. the rest of the text is should remain as is since it's unique to the hero text

content consistency

problem justification solution
the "Digital Communication & Collaboration Fundamentals" and "Implementing Digital Processes & Machine Learning" cards are missing a full stop, all the other cards have it a minor flow issue that users may stumble at when reading the cards sequentially add the missing full stops

Aesthetic issues

consistency & unity

problem justification solution
cards in each section have a different design (border, padding, width, background, heading style, section background colour) unify the card and section styles to one style, only varying colours to create visual separation between cards of different sections. I noticed that the skills section is highlighted with a pink background, which suggests that it's the most important one. I chose not to use it in my redesign but I am not against it. I also borrowed the coloured shading from the hero images for consistency of card-shaped elements. I varied the direction of the shadow between sections to create variety
inconsistent button width/padding (the "Book" button under "Why Learn with our Learning Company?" has a set width while the other buttons' width is determined by text width and padding) this button was removed to remove redundancy, so no aesthetic issues were addressed for it
inconsistent heading styles (some h2's are bold, some aren't) select a heading style to be applied to all headings
heading case difference in the "Specially designed to equip learners with the skills to innovate and succeed" section ("Beginner" vs "INTERMEDIATE" & "ADVANCED") fix capitalisation of headings
inconsistent centering/text alignment between cards in different sections. "Our Digital Learning Solutions" aligns paragraphs to left, other cards center them center all card text
colour scheme is not consistent in use across buttons, section backgrounds, and header backgrounds with seemingly no thought behind it colour schemes are designed with meaning in mind. each colour in a company's branding represents its values. apart from this, colours should have consistent use across elements to reduce confusion. it's common for websites to limit their accents to 1 or 2 colours, unlike this website, which makes it challenging to work with I reserved yellow for buttons and link hover states, blue for helper elements (scrollbar, text field focus), and pink for the asterisks in the forms. I changed the shadow colour of one of the hero images from yellow to blue as yellow is now reserved for buttons. I also thought about using one colour per webpage, visually separating the various services the company offers (e.g. "digital skills" uses the pink accent colour, and "work based learning" uses yellow). both solutions are good, and a matter of personal taste
when hovering over a button, each button colour behaved differently (pink buttons change colour to yellow, yellow buttons lose border but otherwise no changes, yellow button becomes blue) instead of setting a new hover colour for each coloured button, I used darken(colour) to make the button's existing colour darker on mouseover
inconsistent size of blank space between sections "Our Digital Learning Solutions" is closer to the hero section than to the "Why Learn with our Learning Company?" section set the same top and bottom padding for each section
varying paragraph font sizes between sections (e.g. "Our Digital Learning Solutions" has a smaller font than the hero element paragraphs) use the same font size across all paragraphs

contrast

problem justification solution
there are black borders around buttons and images the borders make the webpage appear more cluttered by adding another colour in addition to the 3 accent colours. they also create emphasis on elements that do not need to be emphasised. borders are typically used to create contrast and separation, but the button colours have a good enough contrast with the background that a border is not necessary remove black borders for a cleaner look

placement

problem justification solution
the "Contact Us" button in the hero section overlaps with paragraph text on some screens the user may not be able to read part of the text, which they may find frustrating rewrite the CSS to fix the way the button placement behaves, which can be done with a simple margin property and position:relative instead of absolute

when the browser window is snapped to the side (narrow)

users will often browse the web with 2 (or more) windows displayed on their screen, snapped to different screen areas to facilitate multitasking. this may be the only way they interact with the website and their only impression of it.

problem justification solution
the cards under "Specially designed to equip learners with the skills to innovate and succeed" have misaligned paragraphs, skewing the text to the right and creating extra blank space to the side of it distracts the reader rewrite the CSS to fix the position of the text on narrow screens
there is a very large blank space between the hero "Contact Us" button and the hero paragraph text this may be caused by the way the Wix theme handles image position at different screen sizes I separated the hero section into 2 columns, with each column being in a self-contained wrapper that will not interfere wit each other. on narrower screens, the second wrapper can be moved above the first one, allowing images to be displayed above the text (or removed altogether, with no effect on the rest of the text)
the Twitter and LinkedIn links wrap around the copyright footer text in an unnatural way i wrapped the links into a wrapper so they cannot move away from each other when elements are moved around on narrower screens
